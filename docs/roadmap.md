# Roadmap

Planned improvements organized by time horizon. Items are directional, not committed.

---

## Now

Active priorities for the current period.

- [ ] Add visual documentation to this repository (screenshots of key pages and features)
- [ ] Deepen provider profiles for mid-tier and emerging networks
- [ ] Add new 2026 industry signals as developments occur
- [ ] Improve mobile comparison experience (table readability on small viewports)
- [ ] Strengthen confidence labeling coverage — fill gaps in inferred/estimated fields

## Next

Near-term improvements on the planning horizon.

- [ ] **Additional buyer guides** — expand beyond consulting, PE, and hedge funds to cover legal, corporate development, and government/public sector use cases
- [ ] **Enhanced comparison logic** — add weighted scoring by buyer profile, so "Best for PE" presets reflect PE-specific criteria weights
- [ ] **Provider response tracking** — structured way to note when providers have confirmed or corrected information
- [ ] **Compliance timeline visualization** — timeline view of compliance certifications and regulatory events per provider
- [ ] **AI capability detail pages** — dedicated comparison of AI features (matching, synthesis, Q&A) across providers
- [ ] **Additional VS pages** — expand head-to-head coverage beyond the current 6 matchups
- [ ] **Newsletter or RSS feed** — lightweight mechanism for repeat visitors to track new signals

## Later

Longer-term directions under consideration.

- [ ] **Automated news monitoring** — structured ingestion of provider press releases and industry coverage to supplement manual curation
- [ ] **Expanded provider coverage** — extend beyond the current 33 providers to include smaller regional specialists and emerging entrants
- [ ] **Data visualization** — charts for market share distribution, pricing ranges, compliance maturity progression, and capability heatmaps
- [ ] **API or structured data export** — allow users to access provider comparison data programmatically
- [ ] **Internationalization** — consider multilingual coverage for key markets (APAC, Europe) where expert networks operate
- [ ] **User-contributed corrections** — structured form for users to suggest factual corrections with source documentation
- [ ] **Design system documentation** — formal component library and design token reference for the site's visual language
- [ ] **Performance monitoring** — Core Web Vitals tracking and optimization pipeline

---

## Design Principles for Evolution

As the site evolves, these principles guide what gets added:

1. **User problem first.** Every new feature or page type should address a specific question a real buyer would ask.
2. **Structured over freeform.** Prefer structured data that supports comparison and filtering over narrative prose.
3. **Transparent about certainty.** New content types should integrate with the confidence labeling system.
4. **Maintainable at current scale.** Features should be sustainable for a one-person operation, not dependent on a content team.
5. **Static-first.** Avoid adding server-side dependencies unless there is a compelling reason that static architecture cannot address.
