# SEO and Discoverability

How the site's content and technical structure support search visibility and organic discovery.

---

## Strategic Approach

The site does not rely on paid traffic. Discoverability is designed into the content structure itself — through query-aligned page types, comprehensive topical coverage, and structured data markup.

## Query-Aligned Page Types

Each page type targets a distinct category of search intent:

| Page Type | Target Intent | Example Queries |
|-----------|--------------|-----------------|
| Provider profiles | Brand/navigational | "GLG expert network," "AlphaSights review" |
| VS comparisons | Comparison | "GLG vs AlphaSights," "Third Bridge vs Guidepoint" |
| Alternatives pages | Evaluation | "GLG alternatives," "AlphaSights competitors" |
| Pricing guide | Commercial | "expert network pricing," "how much do expert calls cost" |
| Explainer | Educational | "what is an expert network," "expert network compliance" |
| Rankings | Discovery | "best expert networks," "top expert network providers" |
| Buyer guides | Role-specific | "expert networks for private equity," "expert networks for consulting" |
| News feed | Current | "expert network industry news," "[provider] acquisition" |

This alignment is not accidental. Each page type was designed to serve a specific search intent pattern rather than competing with other pages on the same site.

## Structured Data Implementation

Every page includes JSON-LD structured data:

- **Organization** — site-wide, on every page
- **BreadcrumbList** — on all pages with defined navigation hierarchy
- **CollectionPage + ItemList** — on directory and ranking pages
- **Article** — on comparison pages, buyer guides, and pricing guide
- **FAQPage** — on the explainer page (with 5+ structured Q&As)
- **WebSite** — on the homepage with search action markup

This structured data helps search engines understand page type, content hierarchy, and entity relationships.

## Internal Linking Strategy

The site maintains 8,800+ internal links across 58 pages. Key linking patterns:

- **Profile → Compare:** Provider pages link to the comparison tool with the provider pre-selected
- **Compare → Profile:** Comparison table cells link to provider profiles
- **News → Profile:** Each signal links to its related provider(s)
- **Profile → News:** Provider pages show their most recent related signals
- **Pricing → Profile:** Provider names in the pricing table link to profiles
- **Explainer → Directory/Compare:** Educational pages guide readers toward decision-support tools
- **VS/Alt pages → Profile:** Comparison and alternatives pages link to all mentioned providers
- **Footer → Profile:** Top providers are linked directly from the global footer

This interconnection ensures that link equity flows through the site and that users (and crawlers) can navigate between related content naturally.

## Topical Authority

The site aims to build topical authority through comprehensive category coverage:

- **Breadth:** 33 provider profiles covering the major and notable players
- **Depth:** Detailed comparison data (29 fields), pricing analysis (7 models), compliance tracking, AI capability assessment
- **Freshness:** 57 curated signals tracking industry developments through February 2026
- **Education:** Explainer content, buyer guides, and methodology documentation

Niche authority can emerge from this kind of consistent, structured, and transparent coverage. The site does not need massive domain authority to rank well for specific expert-network-related queries — it needs to be the most thorough and trustworthy resource for this particular topic.

## Technical SEO

- **Static HTML output** — every page is pre-rendered, fully crawlable without JavaScript execution
- **Canonical URLs** — defined on all pages to prevent duplicate content
- **Sitemap** — auto-generated by Astro integration, excluding utility pages (404)
- **Robots.txt** — permissive crawling with sitemap reference
- **Meta tags** — unique title, description, and Open Graph tags per page
- **Page speed** — static assets served from Cloudflare CDN with edge caching
- **Mobile-responsive** — all pages designed for mobile viewports
- **Semantic HTML** — proper heading hierarchy (single H1 per page), landmark regions, and accessible markup

## Known Limitations

- **Domain age** — the site is relatively new, which limits initial search visibility regardless of content quality
- **Backlink profile** — as an independent project, the site does not have enterprise-level backlink acquisition
- **No user-generated content** — no reviews, comments, or community contributions that would add content volume and freshness signals

These are inherent constraints of an independent project. The strategy is to build authority through content quality, topical depth, and structural integrity rather than through scale.
